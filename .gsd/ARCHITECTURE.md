## Components

### Core Layouts
- **Purpose:** Provide the structural and visual framework.
- **Location:** `index.html`, `category.html`, `style.css`
- **Dependencies:** Google Fonts (Outfit), Phosphor Icons.

### Data Management
- **Purpose:** Serve dynamic category listings.
- **Location:** `script.js`
- **Dependencies:** Hardcoded JSON arrays scaling up to 100+ items per category.

### Content Corpus
- **Purpose:** Individual post detail pages generated for every single alert.
- **Location:** `links/*.html` (759+ pages)
- **Dependencies:** Syncing via Python.

### Build Automation scripts
- **Purpose:** Sync structural components iteratively without a modern JS bundler.
- **Location:** `sync_header.py`, `update_footer_all.py`, `refactor_links.py`

## Data Flow

1. **Routing:** User visits `index.html` or direct URL in `/links`.
2. **Global State:** Category filtering processes query strings (`?type=results`), and JavaScript dynamically re-renders lists on `category.html`.
3. **Synchronization:** Developer updates `index.html` and executes Python scripts to propagate `header` and `footer` components sequentially to the `links/` corpus.
